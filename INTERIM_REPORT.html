Executive Summary

This interim report summarizes the progress made on the "Predicting Price Moves with News Sentiment" project. The project aims to enhance Nova Financial Solutions' predictive analytics capabilities by discovering correlations between financial news sentiment and stock market movements. Task 1 (Exploratory Data Analysis) and Task 2 (Quantitative Analysis) have been completed successfully with all required components implemented.

Task 1: Exploratory Data Analysis (Completed)

Objective

Perform comprehensive exploratory data analysis on a large corpus of financial news data (1.4M+ articles) to understand data characteristics, identify patterns, and prepare for sentiment analysis and correlation studies.

Methodology

Data Processing Pipeline

  • Data Loading: Implemented robust data loading functions with validation for 1.4M+ financial news articles
  • Descriptive Statistics: Analyzed textual lengths, publication patterns, and publisher distributions
  • Text Analysis: Extracted keywords, phrases, and performed topic modeling using NLTK and Gensim
  • Time Series Analysis: Examined publication frequency, trends, and timing patterns
  • Publisher Analysis: Identified top contributors, content differences, and unique domains

Tools & Technologies

  • Python 3.11 with pandas, numpy for data manipulation
  • NLTK, TextBlob, WordCloud for text analysis
  • Gensim for topic modeling (LDA)
  • Matplotlib, Seaborn for visualization
  • Jupyter Notebooks for interactive analysis

Date Range: 2009-2023
Publishers: Multiple financial news sources
Key Columns: Headline, Date, Publisher, Stock

Textual Analysis Insights

  • Identified top 50 most frequent keywords across the corpus
  • Extracted significant bigrams and phrases indicating market events
  • Topic modeling revealed major financial themes and market events
  • Word cloud visualization highlighted dominant financial terminology

Temporal Patterns

  • Publication frequency shows clear market activity patterns
  • Identified spikes in publication volume correlating with major market events
  • Publishing times follow business day patterns with peak activity during market hours
  • Seasonal trends observed in financial news coverage

Publisher Insights

  • Identified most contributing publishers and their market share
  • Analyzed content differences between publishers
  • Mapped unique domains and their coverage areas

Deliverables

  • Comprehensive EDA Jupyter notebook with all analyses
  • Modular Python scripts for reusable analysis functions

Progress Status

Status: Complete
Completion: 100%
Data Sources: Local CSV files (AAPL, MSFT, GOOG, META, AMZN, NVDA)

Methodology

Data Preparation

  • Data validation and preparation pipeline ensuring OHLCV completeness
  • Fallback mechanism to yfinance for additional data sources

Technical Indicators (TA-Lib)

  • Moving Averages: SMA (20, 50, 200-day) and EMA calculations
  • RSI: Relative Strength Index (14-day period) with overbought/oversold signals
  • MACD: Moving Average Convergence Divergence with signal line and histogram
  • Bollinger Bands: Upper, middle, lower bands with %B calculation

Financial Metrics (PyNance Concepts)

  • Returns: Daily and cumulative returns calculation
  • Volatility: Rolling and annualized volatility metrics
  • Sharpe Ratio: Risk-adjusted returns calculation
  • Maximum Drawdown: Drawdown analysis and percentage calculations

Visualization

Challenges Encountered

Task 1 Challenges

  • Large Dataset Processing: Handling 1.4M+ articles required optimization techniques including data sampling for computationally intensive operations (keyword extraction, topic modeling)
  • Text Analysis Performance: Implemented sampling strategy (100K rows) for keyword and phrase extraction to balance accuracy and processing time
  • Memory Management: Optimized data loading and processing to handle large text corpora efficiently

Task 2 Challenges

  • TA-Lib Installation: TA-Lib requires pre-compiled binaries on Windows, which can be challenging to install. Implemented robust fallback mechanism using manual calculations with pandas/numpy
  • Data Alignment: Encountered index mismatch issues when plotting returns (due to NaN values from pct_change). Fixed by aligning indices properly
  • Library Dependencies: Some libraries (pandas-ta) had compatibility issues. Resolved by implementing manual calculation fallbacks
  • Data Source Flexibility: Implemented dual-mode data loading (local CSV files and yfinance) to ensure flexibility and reliability

Solutions Implemented

  • Created modular, reusable code with fallback mechanisms
  • Implemented data sampling strategies for large-scale analysis
  • Developed comprehensive error handling and validation
  • Established robust version control and documentation practices
